# Firefox RDP usage

`firefox-lldb` drives Firefox over the DevTools **Remote Debugging Protocol**
(RDP), the same protocol `about:debugging` and the DevTools client use. This
is the full surface we depend on — every actor, request, and event — kept in
sync with the code because it's generated by hand from the single source of
truth, [`src/rdp/protocol.ts`](../src/rdp/protocol.ts). If this document and
that file ever disagree, trust the code and file a bug.

All RDP wire traffic is confined to [`src/rdp/`](../src/rdp/):

- [`transport.ts`](../src/rdp/transport.ts) — length-prefixed JSON framing
  (`<utf8-byte-length>:<json>`) over a TCP socket, matching Firefox's
  `devtools/shared/transport/packets.js`.
- [`client.ts`](../src/rdp/client.ts) — request/reply correlation. Each actor
  processes requests serially (FIFO), so a non-event packet from an actor is
  the reply to that actor's oldest outstanding request. Packets whose `type`
  is a known event (see below) are delivered as unsolicited notifications
  instead.
- [`session.ts`](../src/rdp/session.ts) — the wasm-debugging session: actor
  discovery, thread-config bring-up, breakpoints, stepping, all-stop
  coordination, console/eval.

No other module in the codebase builds or parses an RDP packet.

## Actor graph

All actor ids except `root` are dynamic — assigned by Firefox at connect or
watch time.

| Actor                                     | Obtained from                                                                          |
| ----------------------------------------- | -------------------------------------------------------------------------------------- |
| `root`                                    | the well-known actor of the initial connection                                         |
| preference actor                          | `root`'s `getRoot` reply                                                               |
| tab / target actor                        | `root`'s `listTabs` reply (one per open tab)                                           |
| watcher actor                             | a tab actor's `getWatcher` reply                                                       |
| thread-config actor                       | the watcher's `getThreadConfigurationActor` reply                                      |
| target actor, thread actor, console actor | a `target-available-form` event (one frame target + one per emscripten pthread worker) |
| source actor                              | a thread actor's `sources` reply or a watched `resources-available-array` source form  |
| arrayBuffer actor                         | an `arraybuffer` grip returned by a source actor's `source` reply                      |
| frame actor                               | a thread actor's `frames` reply                                                        |
| longString actor                          | a grip with `type: "longString"`                                                       |

There is no dedicated breakpoint-list actor: `setBreakpoint`/`removeBreakpoint`
are sent per thread actor — the watcher does not broadcast them.

## Requests

| `type`                             | Sent to             | Purpose                                                                                                                                                                                |
| ---------------------------------- | ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `getRoot`                          | root                | fetch the preference actor                                                                                                                                                             |
| `listTabs`                         | root                | enumerate open tabs                                                                                                                                                                    |
| `getCharPref`                      | preference actor    | read a profile pref (used to verify the launch token)                                                                                                                                  |
| `getWatcher`                       | tab/target actor    | get the watcher actor, with `isServerTargetSwitchingEnabled: true`                                                                                                                     |
| `navigateTo`                       | tab/target actor    | navigate the tab, `{ url, waitForLoad }`                                                                                                                                               |
| `getThreadConfigurationActor`      | watcher             | get the actor that accepts thread config                                                                                                                                               |
| `watchTargets`                     | watcher             | subscribe to `target-available-form`/`target-destroyed-form` for `targetType: "frame" \| "worker"`                                                                                     |
| `watchResources`                   | watcher             | subscribe to source resources, `resourceTypes: ["source"]`                                                                                                                             |
| `updateConfiguration`              | thread-config actor | apply `ThreadConfig` (`observeWasm`, `observeAsmJS`, `pauseOnExceptions`, `ignoreCaughtExceptions`) — must be set before navigation so the page's own wasm compiles with debug support |
| `sources`                          | thread actor        | list a thread's sources (wasm + JS)                                                                                                                                                    |
| `frames`                           | thread actor        | list stack frames, `{ start, count }`                                                                                                                                                  |
| `setBreakpoint`                    | thread actor        | set a breakpoint at `{ location: { sourceUrl, line, column }, options }`                                                                                                               |
| `removeBreakpoint`                 | thread actor        | remove a breakpoint at the same `location`                                                                                                                                             |
| `resume`                           | thread actor        | continue, or step with `resumeLimit: { type: "step" \| "next" }` — fire-and-forget, ack arrives as a `resumed`/`paused` event                                                          |
| `interrupt`                        | thread actor        | pause a running thread, `{ when: {} }` — fire-and-forget, ack arrives as a `paused` event (or a `type: "interrupt"` event if already paused)                                           |
| `source`                           | source actor        | fetch source text or wasm bytes                                                                                                                                                        |
| `getBreakpointPositionsCompressed` | source actor        | valid `(line, column)` breakpoint positions in a `{ start: { line }, end: { line } }` range — snapping to one of these is required or the breakpoint silently never fires              |
| `substring`                        | longString actor    | fetch a slice of an over-length source, `{ start, end }`                                                                                                                               |
| `slice`                            | arrayBuffer actor   | fetch base64-encoded browser-owned bytes, `{ start, count }`                                                                                                                           |
| `release`                          | arrayBuffer actor   | release a temporary binary-source actor                                                                                                                                                |
| `getEnvironment`                   | frame actor         | fetch a frame's scope chain (locals, `wasm instance`/`wasm function` scopes)                                                                                                           |
| `startListeners`                   | console actor       | subscribe to `listeners: ["ConsoleAPI", "PageError"]`                                                                                                                                  |
| `evaluateJSAsync`                  | console actor       | evaluate JS, optionally in a frame's scope via `frameActor`; the result arrives later as an `evaluationResult` event carrying the ack's `resultID`                                     |

Full request/response TypeScript shapes: `REQUESTS` and the `*Args`/`*Response`
interfaces in [`protocol.ts`](../src/rdp/protocol.ts).

## Events

Unsolicited notifications, never treated as request replies (`EVENTS` in
`protocol.ts`):

| `type`                                                                                                        | Emitted by       | Meaning                                                                                                                                                               |
| ------------------------------------------------------------------------------------------------------------- | ---------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `target-available-form`                                                                                       | watcher          | a frame or worker target came up                                                                                                                                      |
| `target-destroyed-form`                                                                                       | watcher          | a target went away (tab close, navigation, or a Fission process-swap)                                                                                                 |
| `resource-available-form` / `resources-available-array` / `resource-updated-form` / `resource-destroyed-form` | watcher          | resource lifecycle (only `source` resources are watched); the array form supplies worker actor-to-URL mappings used to symbolize stopped worker frames                |
| `tabListChanged`                                                                                              | root             | the tab list changed                                                                                                                                                  |
| `tabNavigated`                                                                                                | tab/target actor | the tab navigated                                                                                                                                                     |
| `tabDetached`                                                                                                 | tab/target actor | the tab detached                                                                                                                                                      |
| `frameUpdate`                                                                                                 | tab/target actor | frame update (unused beyond routing)                                                                                                                                  |
| `paused`                                                                                                      | thread actor     | the thread paused (breakpoint, step, or interrupt)                                                                                                                    |
| `resumed`                                                                                                     | thread actor     | the thread resumed                                                                                                                                                    |
| `newSource`                                                                                                   | thread actor     | a new source appeared (unused beyond routing)                                                                                                                         |
| `willNavigate`                                                                                                | tab/target actor | navigation is about to start (unused beyond routing)                                                                                                                  |
| `networkEvent`                                                                                                | —                | unused beyond routing                                                                                                                                                 |
| `consoleAPICall`                                                                                              | console actor    | a `console.*` call                                                                                                                                                    |
| `pageError`                                                                                                   | console actor    | an uncaught error or warning                                                                                                                                          |
| `evaluationResult`                                                                                            | console actor    | the async result of `evaluateJSAsync`, matched by `resultID`                                                                                                          |
| `interrupt`                                                                                                   | thread actor     | ACK sent when `interrupt` targets a thread that's already paused or mid-transition — must be classified as an event or it corrupts the next in-flight request's reply |

## Response / event payload shapes

Named interfaces in `protocol.ts`, used directly by `session.ts` (no ad-hoc
casts): `GetRootResponse`, `GetCharPrefResponse`, `RdpTabForm`/`ListTabsResponse`,
`GetWatcherResponse`, `GetThreadConfigurationActorResponse`, `SourceForm`/`SourcesResponse`/`ResourcesAvailableArrayEvent`,
`SourceResponse`/`LongStringGrip`/`ArrayBufferGrip`, `SubstringResponse`,
`ArrayBufferSliceResponse`, `BreakpointLocation`,
`GetBreakpointPositionsResponse`, `FrameForm`/`FramesResponse`, `PauseEvent`,
`StoppedEvent` (our own all-stop event, not a wire form), `EvaluateJSAsyncAck`,
`ConsoleApiCallEvent`, `PageErrorEvent`.
